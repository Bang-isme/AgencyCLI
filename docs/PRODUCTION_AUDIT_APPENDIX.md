## Swarm Model + Agent Registry + Governance

**Maturity: `partial`** — implemented 4 · partial 7 · missing 6

The subsystem implements foundational swarm architecture with worker isolation (snapshots, lock manager, staging engine), cost governance with hard budget ceilings, and provider failover. However, critical governance gaps exist: no recursion depth limits for nested agent delegation, no circular delegation detection, no max_hops enforcement, no agent health/capability registry with dynamic task routing (task assignment is hardcoded by role), no deadlock prevention, and loop detection is only at the inference level, not at the agent orchestration level.

## Task Graph (DAG) + 12-State Machine

**Maturity: `partial`** — implemented 13 · partial 3 · missing 6

AgencyCLI implements a functional DAG execution engine with parallel execution (up to 3 concurrent tasks), proper dependency resolution, and retry logic with recovery strategies. However, the state machine deviates significantly from the spec: only 9 actual states are implemented (PENDING, QUEUED, RUNNING, VERIFYING, RECOVERING, COMPLETED, FAILED, SKIPPED, PAUSED, ABORTED) instead of the specified 12 (missing CREATED, PLANNING, READY, BLOCKED, WAITING_APPROVAL, REVIEWING, CANCELLED). State transitions are enforced via code (TaskStateMachine.transition validates against VALID_TRANSITIONS map), supporting the DAG structure through checkpoint persistence and lease-based worker isolation. Dependency resolution works correctly via areDependenciesMet checks, and tasks can be skipped or marked as aborted. The implementation includes sophisticated recovery strategies (escalating levels 0-6), convergence scoring, stagnation detection, and file-level rollback capabilities.

## Event Sourcing + Execution Journal + Black-Box/Handover

**Maturity: `partial`** — implemented 0 · partial 5 · missing 2

The subsystem has foundational event infrastructure (EventBus, EventJournal, ReplayEngine) with unit test coverage, but lacks critical production wiring. EventBus publishes ~15+ event types (thought:emitted, dag:*, runner:*, approval:*, subagent:*, system:warning) and maintains an in-memory journal, but these events are NEVER persisted to the SQLite EventJournal at runtime. ReplayEngine can validate deterministic playback via hash-matching, but is only used in tests. Execution state checkpoints exist (TaskCheckpoint), but lack correlation with events. No session_summary.md or handover.md generation exists. System is event-aware but not yet event-sourced.

## Memory Architecture (4-Layer) + Governance

**Maturity: `partial`** — implemented 5 · partial 5 · missing 5

The AgencyCLI memory subsystem implements a single-layer SQLite-backed episodic+vector+graph architecture with partial governance controls. Storage persistence and CRDT-based merging are robust; semantic retrieval via RRF, graph integrity checks, and recovery mechanisms exist. Worker write isolation uses a global WriteQueue, but ownership of memory creation is not formally assigned to a MemoryAgent with Orchestrator approval gates.
